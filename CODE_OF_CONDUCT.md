# Code of Conduct

## Our Pledge

We want Kaspa Node Watchtower to be a practical, respectful, and safe project
for people who operate local Kaspa infrastructure. Contributors and maintainers
are expected to communicate with patience, assume good intent, and keep the
project focused on reliable operator tooling.

## Expected Behavior

Examples of behavior that helps this community:

- Use welcoming and inclusive language.
- Give and receive feedback respectfully.
- Keep discussions focused on the technical issue, operational risk, or user
  impact.
- Be clear about assumptions, limitations, and test coverage.
- Respect privacy when discussing logs, configuration, diagnostics, node
  details, or wallet-related context.

## Unacceptable Behavior

Examples of unacceptable behavior include:

- Harassment, threats, insults, or personal attacks.
- Publicly sharing private hostnames, credentials, wallet data, webhook URLs,
  unsanitized logs, or other sensitive operational details.
- Deliberately misleading maintainers or operators about safety, recovery
  commands, alert behavior, or node health.
- Repeatedly derailing project discussions away from constructive technical
  work.
- Any other conduct that maintainers reasonably judge harmful to a respectful
  and safe project environment.

## Scope

This code of conduct applies in project spaces including issues, pull requests,
discussions, commits, release notes, and any project-related communication.

## Enforcement

Maintainers may remove comments, close issues, reject contributions, or limit
participation when behavior violates this code of conduct. Enforcement should
be proportionate to the impact and should prioritize the safety of contributors
and operators.

If a report involves sensitive operational data, follow the private reporting
guidance in `SECURITY.md` and avoid posting secrets or unsanitized diagnostics
publicly.

## Attribution

This code of conduct is adapted from the Contributor Covenant, version 2.1,
with project-specific language for local infrastructure and operational safety.
